# 诊前一步

围绕“家庭成员 + 一次就医事件”整理诊前信息、用户记录的医生说明和后续行动。本仓库当前是依据《用户流程与信息架构（Codex 执行版 V1.0）》建立的第一阶段前端骨架。

本项目不提供诊断、报告解读、用药建议、在线问诊或急诊判断，也不连接医院、医保、支付或真实多人协作系统。数据默认只保存在当前浏览器的 localStorage。

## 本地运行

```bash
npm install
npm run dev
```

质量检查：

```bash
npm run lint
npm run build
```

## GitHub Pages

应用使用 `HashRouter` 和 Vite 相对资源路径。执行 `npm run build` 后，将 `dist/` 发布到 GitHub Pages 即可，无需服务器端路由回退。

## 当前范围

已搭建全部定义路由、统一类型、Context + Reducer 状态、localStorage 容错持久化、虚构演示数据、空/错误状态与响应式布局。患者与事件基础流程、诊前准备、医生说明、行动任务、实时摘要、历史筛选与继承、事件状态迁移、角色脱敏预览，以及住院任务板、陪护交接和出院清单均已可用。摘要支持 A4 打印或另存 PDF；项目已完成移动端、桌面端和 GitHub Pages 静态发布配置检查，但未执行实际发布。
